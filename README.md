# Programmatic AGENTS.md Generation

**An eval-driven approach to creating better AGENTS.md files using transcript-driven hill-climbing.**

## Approach: Transcript-Driven Hill-Climbing

Use eval metrics as the objective function and hill-climb the AGENTS.md. Each iteration produces a smaller, more focused file. Improvements are measured, not assumed.

```
1. INIT      → Generate baseline AGENTS.md via `claude /init`
2. BASELINE  → Run agents (Haiku) with baseline AGENTS.md → get transcripts
3. ANALYZE   → Sonnet reads transcripts, identifies where agents struggled
4. GENERATE  → Sonnet produces targeted AGENTS.md addressing specific failures
5. GRADE     → Deterministic graders: binary exists? imports work? tests pass?
6. COMPARE   → Metrics table: pass@1, tokens, tool calls, wall clock, cost
7. SHRINK    → Sonnet analyzes improved transcripts → even smaller AGENTS.md
8. REPEAT    → Until AGENTS.md size plateaus or pass@1 drops
```

### Key Design Decisions

**Weak agent (Haiku 4.5) for testing.** If AGENTS.md helps Haiku, it helps everything. If Sonnet passes 100% regardless, the eval is useless.

**Strong generator (Sonnet 4.6).** Reading transcripts and identifying failure patterns requires stronger reasoning than the build task itself.

**Doc stripping.** `strip_build_hints.sh` removes all in-tree documentation (README, Doc/, CONTRIBUTING, etc.) so the eval measures AGENTS.md quality, not the agent's ability to find built-in docs.

**JSONL transcripts.** All trials run via `claude -p --output-format stream-json --verbose`. Transcripts are the source of truth — same transcript produces the same metrics, always.

**Shrink each iteration.** Research shows verbose AGENTS.md files hurt performance (arXiv 2602.11988). The final AGENTS.md is 55 words — every line addresses a specific failure observed in transcripts.

A naive approach (pointing a model at the repo without modification) was tried first and discarded — it just parroted in-tree docs into the AGENTS.md, adding zero value. This led directly to the doc-stripping step.

## Defining "Better"

| Priority | Metric | Why |
|----------|--------|-----|
| **P0** | **pass@1** | Did the agent succeed on first try? Non-negotiable. |
| **P1** | **turns** | Fewer turns = agent knew what to do. Most direct signal of guidance quality. |
| **P1** | **wall clock** | User-facing latency. |
| **P2** | **total tokens** | Cost proxy. |
| **P2** | **AGENTS.md size** | Context overhead. Smaller = more room for user context. |

---

## Architecture: Three Roles

| Role | Model | Job |
|------|-------|-----|
| **Evaluator** | Scripts (bash, Python) | Orchestrate trials, run graders, extract metrics from JSONL |
| **Agent** | Haiku 4.5 (weak) | Execute the build task cold inside an isolated worktree |
| **Generator** | Sonnet 4.6 (strong) | Analyze transcripts, identify failures, produce targeted AGENTS.md |

Each trial agent runs in its own git worktree with no access to other trials, evaluator scripts, or metrics. Doc stripping is committed once; every worktree inherits the clean state.

## Evaluation

### Graders

| Grader | Check |
|--------|-------|
| `binary-exists` | `./python` or `./python.exe` exists in worktree |
| `functional-test` | `./python -c "import ssl; import ctypes; import sqlite3; print('BUILD OK')"` |
| `test-suite-smoke` | `./python -m test test_math test_string test_list -v --timeout 60` |

All three must pass for `pass@1 = true`. No partial credit.

### Metrics (extracted from JSONL by `extract_metrics.py`)

`pass@1` · `total_tokens` · `n_tool_calls` · `n_turns` · `duration_ms` · `total_cost_usd` · `n_tool_errors`

## The Final AGENTS.md

**55 words.** Generated by Sonnet 4.6 from baseline transcript analysis — never hand-written.

```markdown
## Build CPython from Source

**Prerequisites are already installed.** Source `build_env.sh` before every build
command — it sets `CPPFLAGS`, `LDFLAGS`, and `CPYTHON_CONFIGURE_EXTRA` (includes
`--with-openssl`) for Homebrew-installed deps.

source ./build_env.sh
./configure $CPYTHON_CONFIGURE_EXTRA
make -j$(sysctl -n hw.ncpu)

Verify: ./python.exe -c "import ssl, sqlite3, readline; print('OK')"

The built interpreter is ./python.exe (not ./python).
```

Every line addresses a specific failure observed in baseline transcripts:

| Line | Failure it prevents |
|------|-------------------:|
| "Source `build_env.sh`" | Agents waste 3–5 turns hunting for dependency locations |
| `$CPYTHON_CONFIGURE_EXTRA` | Agents guess wrong `--with-openssl` path, causing import failures |
| `make -j$(sysctl -n hw.ncpu)` | Eliminates single-threaded builds that 2x wall clock time |
| "`python.exe` not `python`" | macOS build quirk: agents spend 3–5 turns debugging "binary not found" |

## Tradeoffs

- **Single task (CPython build)** — unknown if the approach generalizes to other repos or task types
- **100% baseline pass rate** — the eval measures efficiency, not capability; a harder task would show a starker pass@1 delta
- **Deterministic graders only** — can't evaluate code quality, style, or convention adherence
- **Doc stripping** — agents in production will have those docs; stripping creates a controlled experiment but diverges from real conditions

## Skills + Plugins

While the hill-climbing eval proves a better AGENTS.md helps, **AGENTS.md as a format is a stepping stone**. Skills and plugins offer modular, composable units that load selectively based on task context — no all-or-nothing context consumption. This eval harness itself is packaged as a skill: 13 scripts, 6 commands, 4 reference docs — loaded only when needed.

## Reproducing

### Prerequisites

- [Claude CLI](https://docs.anthropic.com/en/docs/claude-code) · Python 3.9+ · Git · GNU coreutils (`brew install coreutils` on macOS)

### Quick Start

```bash
git clone https://github.com/yourusername/faros-agents-md-eval.git
cd faros-agents-md-eval

# Run the full pipeline (3 iterations, 3 trials each)
bash .claude/skills/agents-md-eval/scripts/hill_climb.sh workspace 3 haiku sonnet 3
```

Or step by step:

```
/eval-init            # Clone CPython, install deps, generate /init baseline
/eval-run baseline    # Run baseline trials
/eval-run improved    # Generate improved AGENTS.md + run improved trials
/eval-metrics         # Display comparison tables
/agents-md-improve    # Hill-climb: shrink AGENTS.md, re-test
```
