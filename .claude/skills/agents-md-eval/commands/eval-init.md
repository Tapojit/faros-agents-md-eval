Set up the eval environment for AGENTS.md evaluation.

This command must work reliably across macOS (local folder), Linux (Claude Code), and Cowork (sandboxed Linux VM). Follow the steps in order. Each step has a verification check — do not proceed if a check fails.

## Steps

### 1. Set skill directory and create workspace

```bash
SKILL_DIR="<path-to-agents-md-eval-skill>"   # adjust to actual installed path
mkdir -p workspace/{repo,agents-md,worktrees,results,eval-suites}
```

### 2. Copy eval suites to workspace

```bash
cp "$SKILL_DIR"/eval-suites/* workspace/eval-suites/
```

**Verify:** `ls workspace/eval-suites/*.yaml` shows the eval suite files.

### 3. Clone CPython (shallow, pinned tag)
```bash
git clone --depth 1 --branch v3.12.0 https://github.com/python/cpython.git workspace/repo
```

**Verify:** `git -C workspace/repo describe --tags` prints `v3.12.0`.

### 3.5. Strip build-hint documentation

Remove in-tree docs that would let agents shortcut the build task (READMEs, Doc/, contributing guides, etc.). This is committed to the repo so every worktree inherits the clean state automatically.

```bash
bash "$SKILL_DIR/scripts/strip_build_hints.sh" workspace/repo
```

**Verify:** `ls workspace/repo/README*` returns no results. `ls workspace/repo/Doc/` fails (directory removed).

### 4. Install system build dependencies

```bash
bash "$SKILL_DIR/scripts/install_deps.sh" workspace
```

The script auto-detects the platform and handles everything. If a non-standard prefix is used, it creates `workspace/build_env.sh` for downstream build steps.

**Cowork setup script (best approach):** If available, put this in Cowork's "Setup script":
```bash
#!/bin/bash
apt-get update -qq && apt-get install -y -qq \
  build-essential libffi-dev libssl-dev zlib1g-dev \
  libreadline-dev libsqlite3-dev libbz2-dev libncurses-dev liblzma-dev bc
```

**Verify:** `bash "$SKILL_DIR/scripts/install_deps.sh" workspace` confirms headers found.

### 5. Generate baseline AGENTS.md via /init

This creates the baseline AGENTS.md — the standard auto-generated file that the hill-climbed version must beat.

```bash
echo "Read through this codebase and generate an AGENTS.md file. Write it to AGENTS.md in the current directory. The AGENTS.md should help a coding agent efficiently build CPython from source. Focus on non-obvious setup steps, build dependencies, configure flags, and common pitfalls." \
  | claude -p --model sonnet --dangerously-skip-permissions --no-session-persistence --tools "Bash,Read,Write" --output-format text \
  > /dev/null 2>&1

# Move it to the agents-md directory
if [ -f workspace/repo/AGENTS.md ]; then
    cp workspace/repo/AGENTS.md workspace/agents-md/init-baseline.md
    rm workspace/repo/AGENTS.md
    echo "Generated init-baseline.md ($(wc -w < workspace/agents-md/init-baseline.md) words)"
fi
```

**Verify:** `wc -w workspace/agents-md/init-baseline.md` shows a word count.

**Note:** The above can also be run automatically by `hill_climb.sh` (Phase 0). If you prefer, skip this step and let the hill-climbing script handle it.

### 6. Initialize history.json
```bash
cat > workspace/history.json <<EOF
{
  "started_at": "$(date -u +%Y-%m-%dT%H:%M:%SZ)",
  "repo": "cpython",
  "task": "build-cpython-from-source",
  "current_best": "baseline",
  "iterations": [],
  "graduated_tasks": []
}
EOF
```

### 7. Initialize metrics.csv with header
```bash
python3 "$SKILL_DIR/scripts/extract_metrics.py" --csv-header > workspace/results/metrics.csv
```

### 8. Sanity-check graders
```bash
bash "$SKILL_DIR/scripts/run_graders.sh" workspace/repo || echo "Expected: graders fail on unbuilt repo (0/3)"
rm -f workspace/repo/grading.json
```

**Verify:** Output says `0/3 passed`.

### 9. Print summary

Display: repo path, tag, build env type, grader status, baseline AGENTS.md word count, available eval suites.

## What NOT to do during init

- **Do NOT build CPython yourself.** Building the repo as the evaluator leaks knowledge and contaminates baseline expectations.
- **Do NOT generate an AGENTS.md manually.** The baseline is generated by `/init` and improved versions come from transcript analysis by `generate_agents_md.py`.
- **Do NOT run any eval trials from init.** Init sets up the workspace. `/eval-run` or `hill_climb.sh` runs the trials.

## After setup

Suggest running the full pipeline:
```bash
bash "$SKILL_DIR/scripts/hill_climb.sh" workspace 3 haiku sonnet 3
```

Or step by step:
```
/eval-run baseline    → runs baseline trials (Haiku, with /init AGENTS.md)
/eval-run improved    → generates improved AGENTS.md from transcripts, runs with it
```

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `sudo: no new privileges` | Cowork sandbox | Use `install_deps.sh` (auto-fallback) or Cowork Setup Script |
| `chmod: Read-only file system` | Skill dir is read-only | Run scripts with `bash`/`python3` — no execute bit needed |
| `claude -p` says "Not logged in" | CLI not authenticated | Run `claude /login` interactively first, or set `ANTHROPIC_API_KEY` |
| `configure: error: Invalid --with-openssl` | Empty env var | Ensure `$DEVPREFIX` is set; re-run `install_deps.sh` |
| `bc: command not found` | Missing in sandbox | Add `bc` to Cowork setup script |
